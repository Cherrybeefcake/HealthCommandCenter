# Exercise Library Import

Health Command Center bundles a normalized offline exercise library generated by `Scripts/import_exercise_library.py`.

## Sources

- `yuhonas/free-exercise-db`
- Source URL: `https://github.com/yuhonas/free-exercise-db`
- License: Unlicense / public domain
- Local license copy: `Docs/Licenses/free-exercise-db-LICENSE.md`

## Imported Data

- 873 general exercise metadata records from `dist/exercises.json`
- 100 HCC-curated resistance-band movements/variations
- 100 HCC-curated mobility, stretch, reset, warmup, cooldown, and recovery movements
- Existing hand-curated HCC records remain preferred when duplicates occur.

## Normalization

The import script maps source fields into `ExerciseDefinition`:

- stable ID
- name and aliases
- category and movement pattern
- primary and secondary muscles
- equipment
- difficulty
- force/mechanic when available
- setup and execution steps
- breathing cue
- common mistakes
- how it should feel
- pain/caution guidance
- substitutions
- location compatibility
- shoulder-friendly and low-back-friendly flags
- source, license, source URL, and import date

## Images

Images from the source dataset are intentionally deferred. The current pass imports text and metadata only. Any future image import should separately audit image licensing, app size impact, and attribution/display needs before bundling assets.

## Runtime Behavior

The app loads `ImportedExerciseLibrary.json` from the app bundle and combines it with existing curated HCC exercises. This keeps search offline and avoids parsing hundreds of raw source files at launch.
